Text Design

<%index %>

You can design text ion various ways with J2ME Polish. NOTE: additional information and examples are pending.

General Design

The mobile Java standard provides three different font-sizes, font-faces and font-styles:

  • size: small, normal/medium, large/big
  • style: normal/plain, bold, italic/cursive, underlined or a mixture of these elements like bold | italic
  • face: system/default, monospace, proportional

Additionally you can specify the color of the font. The following example illustrates the standard font design options for the title style: 

title {
	font-size: large;
	font-face: proportional;
	font-style: bold | underlined;
	font-color: green;
}

Note that on some phones only one font size, one style and one font-type might be supported, like on some (all?) Motorola phones. The system font is an most systems the same as the proportional font.

Bitmap/True Type Fonts

You can convert any true type font to a bitmap font with the font creator tool that you can find in ${polish.home}/bin/fontcreator. After that you can use the bitmap-font by specifying the font-bitmap CSS attribute: 

title {
	font-bitmap: arialBig;
}

Note that you cannot use any other font-setting when you are using a bitmap font.

Text Effects

Bitmap fonts have the drawback that you can only use them in one style, size and color and that you have to add all characters that your application uses into the bitmap. J2ME Polish provides text effects as an alternative solution:

  • shadow: adds a simple shadow to your text.
  • outline: shows the outline of the text only.
  • segment: cuts the text into horizontal slides, similar to the IBM logo.
  • typewriter: animates your text by showing one character more in each animation phase.
  • lighthouse: animates the text by showing one character with its direct neighbours in a different color.
  • gradient: adds a vertical gradient to your text.
  • mirror-shadow: a MIDP 2.0 only effect that adds the vertically mirrored text as a shadow to the original text.
  • fade: You can smoothly fade in or fade our your text on MIDP 2.0 devices.
  • drop-shadow: Adds a soft drop shadow for MIDP 2.0, CLDC 1.1 devices.
  • alien-glow: Use the spooky alien-glow effect for your texts on MIDP 2.0, CLDC 1.1 devices.
  • fading-alien-glow: You can enhance the alien glow even further by fading it in and out.

The following example illustrates the text effect for the title and focused styles: 

title {	
	//#if polish.midp2 and cldc1.1
		text-effect: alien-glow;
	//#else
		text-effect: gradient;
		text-vertical-gradient-start-color: white;
		text-vertical-gradient-end-color green;
	//#endif
	font-size: large;
	font-style: bold;
	font-color: green;
}
focused {
	text-effect: shadow;
	text-shadow-color: red;
}

Note that animated text effects are only animated for the currently focused item, the title and the ScreenInfo element.

Text Wrapping

Usually text is wrapped automatically when it is too long for a single row. You can deactivate this default behavior by setting the text-wrap CSS attribute to false. In TextFields with enabled direct input mode the cursor will be kept in the visible area, for other texts only parts remain visible. You could use this feature to show only the beginning of a message when an item is not focused and show the complete text (by turning on text-wrap again) when an item is focused, for example: 

.messageItem {
	text-wrap: false;
}
.messageItem:hover {
	text-wrap: true;
}
