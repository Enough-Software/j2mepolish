What's in it?

J2ME Polish is a collection of tools for developing J2ME applications:

  • Build tools for creating application bundles out of one source project - for multiple devices and multiple locales.
  • A device database which can be used to adjust your application to different handsets with Preprocessing.
  • A user interface (GUI), which is designed outside of the application code with simple text-files (CSS).
  • Utiliy classes like the BitMapFont or ArrayList.
  • Standalone Tools like the editor for modifying binary data files.
  • ...and this is just the start!

Building J2ME Applications

It's easy to build applications with J2ME Polish:

  • Select your target-devices, either by listing them or by describing their needed capabilities (such as "supports WMAPI and MIDP/2.0").
  • J2ME Polish preprocesses, compiles, obfuscates and packages the application for you.
  • You can also start the respective emulator automatically during the development phase.
  • Since the build tools are based on Ant, J2ME Polish can be configured extensively and integrates nicely into all Java IDEs.

Resource Assembling

  • J2ME Polish includes the appropriate resources for your application automatically.
  • Put common resources into the "resources" folder.
  • You can use specialised resources by putting them into the appropriate subfolder, e.g. "resources/Nokia", "resources/Siemens/S65" or "resources/ScreenSize.160+x200+".
  • It's easy to fine-tune the resource assembling, you can for example include MIDI files only when the device supports MIDI sound, but no MP3 sound and so on.

Localization

  • Localization is needed when the application should be marketed in several regions.
  • You can adjust texts and other resources like images and sounds to different locales easily with J2ME Polish.
  • When you localize your application with J2ME Polish, there will be no overhead compared with non-localized applications because translations are directly embedded into the application.

The Device Database

The Device Database helps to adjust your application to different handsets:

  • Over 300 J2ME devices and their respective J2ME capabilities like Canvas-size, APIs, supported formats etc are listed.
  • You can profit not only by looking into the capabilities, but also by using them directly in your source-code during the preprocessing-phase.
  • Since it is XML based, you can extend it easily.

Preprocessing

  • Preprocessing changes the code before it is compiled.
  • By using preprocessing you can adjust your application to various devices without loosing the portability of your application.
  • J2ME Polish provides you with powerful preprocessing directives and a standarized naming scheme.
  • You can use any capabilities which are defined in the device database.

//#if polish.midp2 || polish.api.mmapi
	// okay the audio-playback of the MMAPI can be used:
	Player player = ...
//#endif

GUI (1/2)

  • J2ME Polish allows you to design the MIDP highlevel-GUI using simple CSS text-files.
  • It's not difficult: background-color: rgb(123, 54, 233);
  • Depending on the item or screen you can use specialized attributes.
  • You can use images, bitmap-fonts, different backgrounds, animations and so on.
  • The complete design is done outside of the application code.
  • So you can easily adjust different "themes" of your application without changing its source-code.
  • With the GUI you can use MIDP/2.0 features like CustomItems on MIDP/1.0 phones!

Game Engine

  • With the Game Engine of J2ME Polish you can use the MIDP/2.0 game API on MIDP/1.0 phones.
  • You can use different optimization strategies like a backbuffer for TiledLayer.
  • The game engine is hightly optimized for speed to make most out of your game.
  • Use one source code for delivering all your target devices.

Utility Classes

  • Use some often needed utilities which are not available in the J2ME standard.
  • de.enough.polish.util.BitMapFont displays messages in a bitmap font.
  • de.enough.polish.util.ArrayList offers a fast alternative for Vector.
  • de.enough.polish.util.TextUtil provides general String functions and splits texts for displaying them on a canvas.

Logging Framework

  • With the logging framework you can view System.out.println()-messages on real devices.
  • You can enable different logging levels like "debug", "warn" or "error" for specific classes or packages.
  • Define your own logging levels like "performance".
  • Deactivate logging completely for the final application, so that absolutely no traces will remain in the application.

try {
	Connection con = Connector.open( url );
	...
} catch (Exception e) {
	//#debug error
	System.out.println("Unable to connect" + e );
}

Standalone Tools

  • Use the binary-editor for creating and modifying binary data like level data files.
  • The font-editor creates bitmap fonts out of any true type fonts.
