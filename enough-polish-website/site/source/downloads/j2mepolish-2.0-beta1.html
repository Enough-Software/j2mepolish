Improvements of J2ME Polish 2.0-beta1

J2ME Polish 2.0-beta1 brings exciting new possibilities to any mobile developer! Please note that the Eclipse Plugin as well as the Java 5 syntax support will be included in the forthcoming Beta 2 release, which is scheduled for mid-August 2006. Please report any problems that you might encounter.

  • UI items: New elements: ListItem for showing lists or tables of items, PictureBrowserItem for showing items, TreeItem for visualizing tree structures like a file system.
  • UI effects: Apply screen-change-animations and text-effects to your design.
  • Advanced Tables: You can stretch elements over more than one table cell by specifying the "colspan" CSS attribute: 
    .singleCell {
	font-color: fontColor;
	background-color: tableCellBackColor;
}
.twoColumnCell extends singleCell {
	colspan: 2;
}
  • IO: Use de.enough.polish.io.Serializable or de.enough.polish.io.Externalizable for making your classes serializable - use de.enough.polish.io.Serializer for writing and reading serializable classes to and from streams.
  • Storage: Use de.enough.polish.io.RmsStorage for saving and loading data to and from the Recordstore Management System. Have a look at the "serialization" sample application for more information.
  • CSS: Use preprocessing within polish.css files just like in your source code. You can even access resources by using property functions. In the following example we specify that an item with the ".mainItem" has at least the same height as the image "button.png". J2ME Polish will then find the correct button.png resource and query its height: 
    .mainItem {
	//#if polish.group.Series60
		background-image: url( button.png );
		min-height: ${ imageheight( button.png ) };
	//#else
		background {
			type: gradient;
			top-color: white;
			bottom-color: red;
		}
	//#endif
}
    New property functions useful for working in polish.css filea are imagewidth, imageheight and exists.
  • Customization: You can now easily customize your application by using several resource roots: 
    
<resources>
	<root dir="resources/base"/>
	<root dir="resources/${customer}"/>
</resources>
    or 
    
<resources>
	<root dir="resources/base"/>
	<root dir="resources/images"/>
	<root dir="resources/sounds"/>

</resources>
  • Internationlization: You can now use messages.txt files with any encoding by using nested <locale> elements: 
    <resources>
	<localization>
		<locale name="fr_FR" encoding="UTF-8"/>

		<locale names="cn, de, en" encoding="UTF-8" unless="test" />
	</localization>
</resources>
  • New backgrounds: There are some new backgrounds available: 
    backgrounds {
	linearGradient {
		type: gradient;
		top-color: white;
		bottom-color: red;
	}
	complexGradient {
		type: gradient;
		top-color: white;
		bottom-color: red;
		start: 5%;
		end: 80%;
	}

}
    By default screen backgrounds range from the very top to the bottom of the menubar. You can change the top and the bottom position for every screen background by specifying the "background-top" and "background-bottom" CSS attributes: 
    
.mainScreen {
	background {
		type: gradient;
		top-color: white;
		bottom-color: red;
		top: title;
		bottom: menubar;
	}
}
  • Subcommands & UiAccess: With de.enough.polish.ui.UiAccess you can access the power of J2ME Polish while staying compatible to the standard javax.microedition.lcdui components. You can, for example, add subcommands to screens by calling UiAccess.addSubCommand( parentCommand, childCommand, screen). Further options include setting the caret position of TextFields, setting the input mode of TextFields and adding ScreenStateListener to any screen.
  • Native placements of menubar elements: When you use the extended menubar, you will get native behavior of the commands, e.g. on Sony Ericsson the default name for the menu will not be "Options" but "Menu" and it will also be placed on the right side of the screen. Activate the extended menubar by setting the "polish.MenuBar.useExtendedMenuBar" property in your build.xml script: 
    <variable name="polish.MenuBar.useExtendedMenuBar" value="true" >
  • Scrollbar: You can use a scrollbar instead of the standard scroll indicators when you activate the polish.useScrollBar preprocessing variable in your build.xml script: 
    <variable name="polish.useScrollBar" value="true" >
    You can then design the scrollbar using the scrollbar CSS style: 
    scrollbar {
	background-color: white;
	scrollbar-slider-width: 5;      /* is ignored when a slider image is defined */
	scrollbar-slider-color: black;  /* is ignored when a slider image is defined */
	scrollbar-slider-image: url( slider.png );
	scrollbar-slider-mode: page; 
		/* The mode of the slider, either "area",
		"item" or "page". In the "area" mode the slider represents the
		position of the current selection relativ to the complete width and height.
		In the "item" mode the index of currently selected item
		is taken into account relative to the number of items.
		In the "page"
		The default mode is "area", unless there is no focusable item
		on the screen, in which case the "page" mode is used.
		*/
	
}
  • Texteffects: You can apply effects to any StringItems, titles, menu items, etc - without having a bitmap font or any additional resources in your JAR!
    • shadow: adds a simple shadow to your text.
    • outline: shows the outline of the text only.
    • segment: cuts the text into horizontal slides, similar to the IBM logo.
    • typewriter: animates your text by showing one character more in each animation phase.
    • lighthouse: animates the text by showing one character with its direct neighbours in a different color.
    • gradient: adds a vertical gradient to your text.
    • mirror-shadow: a MIDP 2.0 only effect that adds the vertically mirrored text as a shadow to the original text.
    • fade: You can smoothly fade in or fade our your text on MIDP 2.0 devices.
    • drop-shadow: Adds a soft drop shadow for MIDP 2.0, CLDC 1.1 devices.
    • alien-glow: Use the spooky alien-glow effect for your texts on MIDP 2.0, CLDC 1.1 devices.
    • fading-alien-glow: You can enhance the alien glow even further by fading it in and out. 
    title {	
	//#if polish.midp2 and cldc1.1
		text-effect: alien-glow;
	//#else
		text-effect: gradient;
		text-vertical-gradient-start-color: white;
		text-vertical-gradient-end-color green;
	//#endif
	font-size: large;
	font-style: bold;
	font-color: green;
}
focused {
	text-effect: shadow;
	text-shadow-color: red;
}

Improvements of J2ME Polish 1.3-beta3

J2ME Polish 1.3-beta3 features bug fixes as well as some big improvements for the 1.3-beta2 release. Check out this release now!

Bugfixes since 1.3-beta2

  • TabbedForm: pointer events are now processed normally.
  • ScrollIndicator: in some cases the scrollindicator was painted above the menu bar.
  • POPUP ChoiceGroups : such ChoiceGroups are now handled correctly when a pointer event occurs while the group is opened. Also horizontal alignments are done correctly now.
  • Locale.java: The Locale.java class is now always included.
  • CSS: styles that have a complex hierarchy are now correctly inherited thanks to Eugene Markov.
  • LogViewer: The logviewer sample application now contains the correct resources.

Improvements since 1.3-beta2

  • Smooth Scrolling: Screens are now scrolled "smooth" by default. You can revert to the previous behavior by specifying scroll-mode: normal; in the Screen's style definition in the polish.css file.
  • DoJa: J2ME Polish now supports the Japanese DoJa standard out of the box.
  • Resource Assembling:
    1. Excluding files: You can now exclude specific resources either by placing files with a length of 0 into the resources tree or by using the <filter> element within the <resources> element.
    2. Enhanced filesets: It's possible to use any J2ME Polish preprocessing variables within <fileset> elements now: 
      <fileset 
	dir="resources/multimedia/images/${polish.language}" 
	includes="*.png" 
/>
  • Alert: You can now design alerts just like other screens. Use the predefined styles alert and alertcontent for designing them or use your own CSS styles using the //#style preprocessing directive.
  • Optional Styles: You can now mark a style as optional by ending the name with a question mark within the #style directive. When the style is defined, it will be included normally, otherwise it will not be used: 
    //#style mySpecificStyle?
Form form = new Form( title );
  • NetBeans: J2ME Polish now supports NetBeans 5.x out of the box. Just open any of the provided sample projects, rename the build-netbeans.xml script to build.xml and you're done. Also compile errors are now mapped to the original source files rather than the preprocessed ones.

Changes since 1.2.4

  • Screen Change Effects: You can now enable effects that are played whenever the current screen is changed. This is demonstrated by the "roadrunner" sample application, just register an effect by setting the screen-change-animation CSS attribute of the corresponding screen: 
    screen-change-animation: verticalflash;
    These effects are described in detail in the Pro J2ME Polish book.
  • BlackBerry and Palm Support: BlackBerry and Palm platforms are now supported out of the box. You can easily adjust J2ME Polish to other platforms, too.
  • Direct Input in DateFields: You can now activate the direct input mode of DateFields by setting the "polish.DateField.useDirectInput" variable in your build.xml file: 
    <variable name="polish.DateField.useDirectInput" value="true" />
  • SpriteItem: use the new SpriteItem for creating sleazy animated menus.
  • MessageItem: use the new MessageItem for showing a combination of a IconItem (for the headline along with an icon) and a StringItem (for the text).
  • ScreenInfo: you can now show an information item on every screen with the new ScreenInfo element. Additionally you need to activate this info element by setting the "polish.ScreenInfo.enable" preprocessing variable to "true" in your build.xml file: 
    <variable name="polish.ScreenInfo.enable" value="true" />
  • Titles: you can now easily apply a new style to the title of a screen by calling the Screen.setTitle() method along with the //#style directive: 
    if ( this.isRegistered ) {
	//#style registeredTitle
	this.form.setTitle( "Hello World" );
} else {
	//#style normalTitle
	this.form.setTitle( "Hello World" );
}
  • FramedForm: You can split your screen into several areas with the new FramedForm. It behaves just like Form, but the frames will always stay in their original positions. Ideal for all pen-based devices and for applications that show information and allow the user to influence that information, e.g. dictionaries with an integrated search field on the result screen.
  • ImageBackground: Using the anchor CSS attribute resulted in a compile error. You can now tile images much more finegrained using the overlap, padding-horizontal and padding-vertical CSS attributes for the image background.
  • exclusive view-type: you can now use the "exclusive" view-type for exclusive ChoiceGroups. This view only shows the currently selected item and allows to choose other items by using the left and right arrows.
  • TextField: You can now deactivate the flashing of the cursor by specifying textfield-caret-flash: false;.
  • abortOnError/onError: J2ME Polish now continues to build even when an error occurs for one device. You can specify that the complete build should be aborted by setting the "abortOnError" attribute of the <build> element. <br /> You can also call an Ant target automatically when an error occurred by specifying the target's name in the &quot;onError&quot; attribute of the &lt;build&gt; element. When an error occurs, J2ME Polish sets the Ant property "j2mepolish.error" to "true" and the "j2mepolish.error.message" property contains the error message. 
    	<build abortOnError="false" onError="handleErrorTarget">
		...
	</build>
	
	...
<target name="handleErrorTarget">
	<echo message="${j2mepolish.error.message}" />
</target>
  • Preprocessor: <preprocessor> elements can now use preprocessing terms as if- or unless-conditions.
  • Obfuscator: <obfuscator> elements can now use preprocessing terms as if- or unless-conditions.
  • Source: <source> elements can now use preprocessing terms as if- or unless-conditions. This allows you for example to use different base classes without using preprocessing in your source code.
  • Third Party Libraries: Thanks to the new <libraries> and <library> elements, you can now use different third party libraries depending on conditions: 
    <libraries>
	<library file="thirdparty/tinylines60.zip" 
	         if="polish.group.Series60 and not polish.midp2" />
	<library file="thirdparty/tinylinemidp2.zip" if="polish.midp2" />
</libraries>
  • Copier: The <resources> element now support the setting of <copier> elements that are responsible for copying the resources after they have been assembled. The following examples removes all characters that are surrounded by curly braces in resource names. The resource bg{background-Series60}.png will be renamed to bg.png, for example: 
    <resources
	dir="resources2" 
	defaultexcludes="yes"
	excludes="*.db"
 >
	<copier name="renamer">
		<parameter name="searchPattern" value="\{.*\}" />
		<parameter name="replacement" value="" />
	</copier>
	<copier name="svgconverter /">
</resources>
  • Extend all build phases: You can now enhance and adjust your build process by using the improved and new extensions like <preprocessor>, <precompiler>, <compiler>, <postcompiler>, <obfuscator>, <copier>, <packager> and <finalizer>. Learn the details in the Pro J2ME Polish book.
  • TextUtil: Strings with hardcoded linebreaks were not processed correctly on Series 40 devices, since the Font.stringWidth()-method behaves oddly on those devices.
  • columns-width: Using a star in the columns-width CSS attribute is now resolved during runtime which results in more sensible settings.
  • Known Issues Database: The known issues database is now better accessible.
  • Dynamic Styles: Using dynamic styles along with a ChoiceGroup in a Form resulted in a NullPointerException. Please note: it is recommended to use only static styles.
  • TabbedForm: Using a tab without any focusable items resulted in an IndexOutOfBoundsException; now you can query the currently selected tab by calling tabbedForm.getSelectedTab(). You can be notified about any changes of the tab by registering a de.enough.polish.ui.ScreenStateListener.
  • focused-style: You can now use a different focused-style for every item. So you can now have a general focused-style, a screen-specific focused-style and an item-specific focused-style. This is useful when you have a button in a Form or when you have TextFields along with other items in one Form, for example. 
    //#style .input
TextField field = new TextField( "name:", null, 20, TextField.ANY );
//#style .button 
StringItem button = new StringItem( null, "Go!", Item.BUTTON );
button.setDefaultCommand( new Command("Go", Command.ITEM, 1 ) );
button.setItemCommandListener( this );

    .input {
	font-color: white;
	background-color: black;
	
	focused-style: .inputFocused;
	
}
.inputFocused extends .input {
	background {
		color: white;
		border-color: black;
		border-width: 2;
	}
	font-color: black;
}
.button {
	background {
		type: round-rect;
		color: red;
		border-color: black;
		border-width: 2;
	}
	font-color: black;
	
	focused-style: .buttonFocused;
	
}
.buttonFocused extends .button {
	background-color: yellow;
	background-border-color: blue;
}
  • min-width, max-width: any items can now be limited in their size by using the CSS attributes min-width and max-width. This is especially useful when you want to align labels. The following example uses the max-width along with the "expand" layout to bring all labels to the same width. The "label" style is predefined by the way and is used by default for all labels. 
    //#style .input
TextField field = new TextField( "name:", null, 20, TextField.ANY );

    label {
	font-color: black;
	font-size: small;
	
	max-width: 60;
	layout: expand;
	
}

.input {
	font-color: white;
	background-color: black;
	focused-style: .inputFocused;
	
	max-width: 80;
	layout: expand;
	
}
.inputFocused {
	background {
		color: white;
		border-color: black;
		border-width: 2;
	}
	font-color: black;
}
  • BitMapFont: the method getFontHeigth() is now renamed to getFontHeight()
  • polish.Container.allowCycling: You can now use the polish.Container.allowCycling preprocessing variable for allowing Forms, Lists, menus etc to be cycled through by the user. When the variable is "false", the user won't be able to scroll from the first item to the last item and vice versa anymore. By default this is allowed. 
    <variable name="polish.Container.allowCycling" value="true"/>
  • Additional APIs: J2ME Polish now also provides the optional libraries FileConnection, PIM-API and Scalable 2D Vector Graphics.
  • LogHander: You can now use the RMS loghandler for storing all log entries into the recordstore: 
    <debug level="error" if="test">
	<loghandler name="rms" />
	<filter package="com.company.package" level="debug" />
</debug>
  • #abort preprocessing directive: You can use the new //#abort directive for stopping the preprocessing in case you have encountered a misconfiguration, e.g. 
    //#if !config.ServerUrl:defined
	//#abort You need to set the config.ServerUrl preprocessing variable!
//#endif
  • new property functions: The new number property function counts the number of different values that are separated by commas within one preprocessing variable: 
    //#= String[] formats = new String[ ${ number( polish.SoundFormat ) } ];
int i = 0;
//#foreach format in polish.SoundFormat
	//#= formats[i] = "${format}";
	i++;
//#next format

    The classname eases the usage of the "useDefaultPackage" option of the <obfuscator>. When the option is activated classname returns only the name of the class without the package, when the "useDefaultPackage" option is not activated, the fully qualified name of the class is returned: 
    //#if SoundListenerImplementation:defined
	//#= ${ classname( SoundListenerImplementation ) } listener;
//#else
	SoundListener listener;
//#endif

    The replace function can replace parts of the processed String, e.g. 
    //#= String url = "${ replace( config.ServerUrl, www, www1 ) }";
  • Automatic inclusion of preprocessing variables and Ant properties: You can now use Ant properties and any preprocessing variables without using the //#= directive, after you have activated the "replacePropertiesWithoutDirective" option in your build.xml: 
    <variables 
	includeAntProperties="true"
	replacePropertiesWithoutDirective="true"
>
	<variable file="cfg/${ lowercase(polish.vendor) }.properties" />
</variables>
    You can then use any properties and property functions directly: 
    	System.out.println("Using device ${ polish.identfier }..." );
  • Jar, KZip and 7-Zip: Jar, KZip and 7-Zip are now supported "out of the box", when these packagers are either on the PATH or in ${polish.home}/bin: 
    <packager name="jar" />
<packager name="7zip" />
<packager name="kzip" />
  • Signing MIDlets: You can now use the <sign> element for signing MIDlets automatically. The signing is done for MIDP/2.0 devices only: 
    <sign
	key="SignMIDlet"
	keystore="midlets.ks"
	password="${password}"
	unless="test"
/>
  • InitializerSplashScreen: You can use the de.enough.polish.ui.splash.InitializerSplashScreen for displaying a splash screen while letting your application initialize in the background.
