Improvements of J2ME Polish 1.2.3c

J2ME Polish 1.2.3c just fixes the broken "useDefaultPackage"-option of the <obfuscator> element. In case this option makes trouble in your project, please make sure that you only use import-statements rather than using classes fully qualified.

This example breaks (don't do it like this): 

	public class MyClass extends com.company.package.MyOtherClass {

This example works fine (do it like this): 

	import com.company.package.MyOtherClass;
	
	public class MyClass extends MyOtherClass {

Improvements of J2ME Polish 1.2.3b

J2ME Polish 1.2.3b is a small update with following bugfixes and improvements:

  • sample build.xml: The "useDefaultPackage"-attribute of the <obfuscator> element was spelled wrong in the sample application of the 1.2.3a release.
  • Display.setCurrentItem( Item ): When Display.setCurrentItem( Item ) was called when the corresponding screen was never shown before, the focused item did not always get the correct CSS-style.
  • TextField: The PASSWORD TextField did not work properly when a default value had been given and the direct input mode was used.
  • Motorola/Non-Fullscreen-Mode: When the GUI was used for Motorola devices and the fullscreen mode was deactivated, a compile error occurred.
  • Sony-Ericsson P910/Fullscreen-Mode: The Sony-Ericsson P910 does crash, when Canvas.setFullScreenMode() is called somewhere else than in the paint()-method. This bug is now circumvented by the J2ME Polish GUI.
  • Device Database: The device database now offers a user friendly view of the most important capabilities and known issues.
  • Thanks: for this release go to Philipp Stolze, Salvatore Pellegriti, Milosz Weckowski and Wuyad.

Improvements of J2ME Polish 1.2.3a

J2ME Polish 1.2.3a is a small update with following bugfixes and improvements:

Bug Fixes

  • Automatic Stacktrace Resolving: Under Windows the automatic resolver of stacktraces got stuck when the jad decompiler issued more than 40 lines of errors.
  • polish.TextField.useDirectInput: Setting that preprocessing variable resulted in a compile error.
  • compilerModePreverify: The preverifying in the compiler-only mode did not work correctly.
  • Form.set(): Form.set() and Form.add() did not trigger a repaint.

Improvements

  • Obfuscator.useDefaultPackage: J2ME Polish can now move all classes to the default package for you. This includes the MIDlet-classes as well and can result in a smaller jar-size. Please note that you are responsible for preventing name-clashes of classes. You do not need to adjust your <midlet> settings, this is done automatically by J2ME Polish. 
    <obfuscator name="ProGuard" useDefaultPackage="true" unless="test" />
  • New Borders: you can now use the additional border-types "bottom", "top", "left" and "right", which support the same attribute as the "simple" border, for example: 
    focused {
	font-color: red;
	border {
		type: bottom;
		color: red;
		width: 2;
	}
}
  • Display.setCurrentItem( Item ): Display.setCurrentItem( Item ) is now automatically resolved by J2ME Polish and can be used for MIDP/1.0 as well as MIDP/2.0 phones if you use the J2ME Polish GUI.
  • <obfuscator> and <debug>: The <obfuscator> and <debug> elements do not need the "enable" attribute any more - this is now always true and is deprecated. Please use the "if" and "unless" attributes in case you need to control those elements from now on.
  • Thanks: for this release go to Andrew Emmons, Thomas Kraft, Milosz Weckowski, Philipp Stolze, Eran and Fernando Gomes.

Improvements of J2ME Polish 1.2.3

J2ME Polish 1.2.3 brings a full screen UI for all MIDP/2.0 phones, the ability to automatically resolve stack traces like "RuntimeException at at com.company.MyClass.mymethod(+22)", an editor for binary files like level-data, the ability to use different packagers, and provides many small but fine improvements, a couple of bug fixes and an updated device database as well as documentation:

  • Bitmap Fonts: Using the font-editor in the "${polish.home}/bin" folder, you easily create bitmap-fonts out of any True Type font and use them in your application. Within the GUI you can use bitmap-fonts by setting the "font-bitmap"-attribute in the "polish.css"-file: 
    .myStyle {
	/* Use the coolfont.bmf for this style: */
	font-bitmap: coolfont;
	background-color: green;
}
    You can also use bitmap fonts independently of the J2ME Polish GUI by using the BitMapFont-class.
  • Fullscreen-Mode: The full-screen mode is now supported for MIDP/2.0 devices as well. The mode can be set with the "polish.FullScreen"-variable as well, which allows a finer grained control than using the "usePolishGui"-attribute of the <build> element: 
    <build
	usePolishGui="true"
	fullscreen="menu"
	>
	<variables>
		<variable 
		name="polish.FullScreen" 
		value="yes"
		if="polish.vendor == Siemens"
	/>
	<variables>
	[...]
  • Automatic Stack Trace Resolving: When an application error occurs during an emulator-session, J2ME Polish can decompile and pinpoint the corresponding source-code-location directly from error messages like "java.lang.RuntimeException at com.company.MyClass.mymethod(+22)". For this J2ME Polish uses the "jad"-decompiler, which is available at http://www.kpdus.com/jad.html. This is described in more detail in the emulator documentation.
  • Packaging: J2ME Polish can now use different packagers as well, which can be specified with the <packager> element: 
    <packager
 executable="jar"
 arguments="cvfM;;${polish.jarPath};;-C;;${polish.packageDir};;."
/>
  • Binary Data Editor: J2ME Polish now includes an editor for binary data files, e.g. level data. Have a look at the "${polish.home}/bin"-directory.
  • Direct Input Mode: You can activate a direct input mode for textfields by specifying textfield-direct-input: true; in the "polish.css" file now.
  • Sony Ericsson Emulators: Emulators from Sony Ericsson can now be launched automatically by J2ME Polish, even when they are not integrated in the "normal" WTK.
  • Lists: In Lists you can now show only the images and use the appended texts as titles by setting the "show-text-in-title"-CSS-attribute to "true".
  • Screens: Improved the layout of "normal" tables, in which each column can use as much space as needed (and possible).
  • Scrolling Indicator: The scrolling indicator is now painted on the left bottom for Siemens devices by default.
  • Directory Handling: J2ME Polish can now handle absolute paths for the directories as well, e.g. <property name="dir.work" value="C:\temp\polish" />.
  • Obfuscators: J2ME Polish now includes the yGuard obfuscator as well and also supports the "KlassMaster" and "Dasho" obfuscators.
  • Game Engine: Corrected an error when a sprite is created using another sprite while Nokia's UI API is available. Also improved the performance of Sprite and LayerManager.
  • GUI Commands: Added commands "mark", "unmark" and "select" to Lists and ChoiceGroups. Of course these can be localized as well. They can be deactivated by defining the <variable>s: 
    <variable name="polish.ChoiceGroup.suppressMarkCommands" value="true"/>
<variable name="polish.ChoiceGroup.suppressSelectCommand" value="true"/>
<variable name="polish.TextField.suppressCommands" value="true"/>
  • Preprocessing: You can now use the new "!="-operator for comparing values: 
    //#if polish.vendor != Nokia
	// this is not a Nokia device...
//#endif
  • Libraries: The Thread.MIN_PRIORITY had the wrong value 0 instead of 1 in the included midp2.jar library. J2ME Polish now also includes the Mobile 3D Graphics API (m3g.jar) as well as the bluetooth API (btapi.jar).
  • Device-Selection: All requirements now support the "undefined" value, which means that the specified capability must not be defined: 
    <deviceRequirements>
	<or>
		<requirement name="MaxJarSize" value="148+ kb" type="Memory"/>
		<and>
			<requirement name="StorageSize" value="148+ kb" type="Memory"/>
			<requirement name="MaxJarSize" value="undefined" type="Memory"/>
		</and>
	</or>
</deviceRequirements>
    When a generic deviceRequirement-type like "Memory", "String", "Int" or "Version" was used, the name and the value of the requirement got mixed up.
  • Thanks: Thanks for this release goes to Andrew Barnes for making the direct input mode of TextFields possible, Thomas Kraft for reviewing the forthcoming J2ME Polish book, Piotr Czarny, Jon Malmquist, Milosz Weckowski and Thomas Broyer for extensive beta testing, providing excellent bug reports and corrections to the device database.
