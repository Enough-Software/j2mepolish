variables for pre-processing

polish knows several variables and switches which can be set directly or indirectly in the polish.xml file. All switches and variables start with "polish.", so you should not encounter any difficulties with your own switches or variables.

This documented is targeted at developers who wants to use polish-specific variables or switches in their own widgets or canvasses.

Switches

Switches are used to indicate specific properties. They are often mutually exclusive with others.

  • polish.midp1: this switch is enabled when the current device is an MIDP1-device.
  • polish.midp2: this switch is enabled when the current device is an MIDP2-device.
  • polish.supportPolishGui: is set when the current device is capable of using the extended GUI of the polish framework.
  • polish.images.backgroundLoad: this switch is set when images of the widgets, backgrounds and canvasses should be loaded in a background thread. This can speed up the responsiveness of an application dramatically, but it also means that some images are set while the user is in a screen. This switch is mutually exclusive with the switch polish.images.directLoad. This switch is set indirectly in polish.xml by the imageLoadStrategy-tag.
  • polish.images.directLoad: this switch is set when images of the widgets, backgrounds and canvasses should be loaded in the foreground thread. This can slow down the responsiveness of an application, but it also means that all images are loaded when the user enters a screen. This switch is mutually exclusive with the switch polish.images.backgroundLoad. This switch is set indirectly in polish.xml by the imageLoadStrategy-tag.
  • debug: when this switch is set, debug information and errors will be printed on System.out and System.err.
  • polish.debug.visual: when this feature is enabled, all debugging info will be transfered to a canvas.
  • polish.supportsPolishGui: defines whether the polish-GUI is supported by the current device.
  • polish.api.[api-name]: defines whether the given api is supported by the current device. You can check for the support of the multi media api with this statement:
    //#ifdef polish.api.mmapi
    Note: all api-names will be written in lowercase, regardless how they were written in the devices.xml. This is not the when you check for JavaPackage - then the api will have the same case as in the devices.xml file.
  • polish.[capabilty-name]:defined: for each defined capability a symbol with an apended ":defined" is defined. You can check if the capability "TextSelectionColor" has been defined with
    //#ifdef polish.TextSelectionColor:defined
  • polish.[capabilty-name].[capability-value]: you can check for any specific capability-value with this switch. For example you could check if the current device supports the https-protocol with
    //#ifdef polish.JavaProtocol.https

Variables

Variables hold specific values.

  • polish.source: always points to the current source directory of a preprocessed file. This can be usefull for #include directives.
  • polish.name: The name of the current device.
  • polish.vendor: The name of the vendor of the current device.
  • polish.identifier: The name of the current device in the format [vendor]/[model], e.g. "Nokia/3650".
  • polish.JavaPlatform: the platform of the current device, e.g. "MIDP/1.1"
